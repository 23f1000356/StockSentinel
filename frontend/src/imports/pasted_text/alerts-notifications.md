Design a modern SaaS **Alerts & Notifications Page** for an Inventory Management System. The page should display real-time alerts like low stock warnings and subscription expiry notifications.

---

🎨 THEME & VISUAL STYLE:

Use a premium SaaS design system:

* Primary Accent: #C89B5A (gold)
* Background: #F5F1EB (soft beige)
* Sidebar: #1E1E1E (dark)
* Cards: #FFFFFF
* Text Primary: #1F2937
* Text Secondary: #6B7280

Design guidelines:

* Minimal, clean layout
* Rounded corners (rounded-xl or 2xl)
* Soft shadows (shadow-md)
* Spacious padding (16–24px)
* Use icons for clarity (🔔 ⚠️ ⏳ ❌ ✔)

---

🧩 LAYOUT STRUCTURE:

* Left Sidebar (dark)
* Top Navbar
* Main Content Area

---

📌 1. LEFT SIDEBAR

* Dark background (#1E1E1E)
* White text
* Active item highlighted in gold (#C89B5A)

Menu:

* Dashboard
* Products
* Warehouses
* Analytics
* Import
* Export
* Alerts (active)
* Activity Logs
* Users
* Subscription
* Logout

---

🔝 2. TOP NAVBAR

* Background: #F5F1EB
* Left: Search bar
* Right: Notification bell icon + Profile avatar

---

🔔 3. PAGE HEADER

* Title: "Alerts & Notifications"
* Subtitle: "Monitor system alerts and important updates"

---

📊 4. ALERT SUMMARY CARDS

Display 4 cards in a row:

1. Total Alerts → 8
2. Low Stock Alerts → 5 ⚠️
3. Subscription Alerts → 2 ⏳
4. Critical Alerts → 1 ❌

Design:

* White cards
* Rounded corners
* Soft shadows

---

🚨 5. FILTER TABS

Horizontal tabs:

[ All ]   [ Low Stock ]   [ Subscription ]   [ Critical ]

* Active tab highlighted in gold
* Others in neutral gray

---

📋 6. ALERT CARDS (MAIN SECTION)

Use card-based layout (not table).

---

Content:

* Timestamp: "Today"

Action:

✔ RESOLVED ALERT (OPTIONAL)

Each alert should show:

🚦 11. UI STATES

* Few alerts visible

WARNING:

* Multiple low stock alerts
* Yellow highlights

CRITICAL:

* Subscription expired

---

🎨 12. DESIGN DETAILS

* Alert cards should have:

  * Icon on left
  * Content center
  * Actions on right

* Use subtle hover effects

* Maintain consistent spacing

* Keep UI uncluttered

---

🎯 GOAL:

The UI should clearly demonstrate:

* Notification system functionality
* Real-time alerts (low stock + subscription)
* Actionable alerts (not passive)
* Clean SaaS user experience
